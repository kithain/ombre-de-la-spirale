FEUILLE DE ROUTE UNIFIEE - "L'OMBRE DE LA SPIRALE"
Wiki de Campagne JDR (React/Vite)
================================================================================

OBJECTIF GENERAL
----------------
Fusionner les meilleures propositions des équipes A et B pour :
- Assainir l’architecture (sortir du composant monolithique CampaignWiki.jsx)
- Mettre en place un routing moderne et une UI réutilisable
- Ajouter des fonctionnalités utiles au MJ (recherche, mode MJ/Joueur, notes, tracker)
- Améliorer l’UX, l’accessibilité et la performance

STRUCTURE GLOBALE DES PHASES
----------------------------
PHASE 1 : Squelette propre & refactorisation (Architecture / Routing / UI de base)
PHASE 2 : Etat & fonctionnalités essentielles pour le MJ
PHASE 3 : UX / UI & Accessibilité
PHASE 4 : Fonctions avancées MJ & enrichissement du contenu
PHASE 5 : Performance, PWA & export

PHASE 3 – UX / UI & ACCESSIBILITE
================================================================================

Objectif : améliorer la fluidité d’usage et respecter les bonnes pratiques d’accessibilité.

3.1. Navigation & ergonomie
---------------------------
- Intégrer le Breadcrumb (Breadcrumb.jsx) dans les vues concernées :
  - Exemple : Univers > Zone > Lieu > PNJ

- Responsive mobile :
  - Remplacer les simples onglets par :
    - un menu burger pour la navigation globale
    - ou des tabs adaptés sur petit écran (sticky, scroll horizontal, etc.)

- Raccourcis clavier :
  - Navigation entre onglets / sections :
    - Flèches gauche/droite ou J/K
  - Documenter ces raccourcis (ex : petite aide dans le header ou footer)

3.2. Indicateurs visuels & progression
--------------------------------------
- Scénario en cours :
  - Visuel distinct (bordure, tag “En cours”, surbrillance) dans ScenariosView.

- Compteur de scènes par acte :
  - Affichage du nombre de scènes dans ActAccordion.jsx

- Barre de progression par scénario :
  - Basée sur (scènes jouées / scènes totales)
  - Affichage dans ScenarioRow.jsx ou dans la page de détail du scénario

3.3. Animations & transitions
-----------------------------
- Créer /src/styles/animations.css :
  - Définir les animations CSS manquantes :
    - fadeIn
    - slideDown
    - etc.

- (Optionnel) Intégrer framer-motion :
  - Transitions de pages (changement de route)
  - Animation d’ouverture/fermeture des accordéons
  - Animations légères sur les cartes (Card.jsx) au survol

3.4. Accessibilité (A11y)
-------------------------
- Ajouter les attributs ARIA :
  - Sur les boutons, tabs, accordéons, menus, etc.
  - Exemples : aria-expanded, aria-controls, role="tab", role="tablist", etc.

- Navigation clavier complète :
  - Tous les éléments interactifs atteignables via Tab / Shift+Tab
  - Ordre de focus cohérent

- Contraste :
  - Vérifier que les couleurs clés respectent les contrastes recommandés (WCAG)

- Focus visible :
  - Styles de focus clairs et visibles sur tous les éléments interactifs

PHASE 4 – FONCTIONS AVANCEES MJ & ENRICHISSEMENT DU CONTENU
================================================================================

Objectif : transformer l’outil en véritable “écran du MJ”.

4.1. Outils avancés pour le MJ
------------------------------
- Vue “Écran du MJ” / Dashboard :
  - Vue synthétique pour la session en cours :
    - Scénario actuellement joué
    - Prochaines scènes prévues
    - PNJ clés à proximité
    - Rappels des éléments importants (indices, menaces, etc.)

- Timeline interactive de la campagne :
  - Ordre chronologique des scénarios et des scènes majeures
  - Possibilité d’afficher les événements importants et leurs conséquences

- Tracker de progression global :
  - Indicateur de progression par scénario et pour la campagne globale

4.2. Contenu & médias
----------------------
- Vérifier la cohérence des 7 zones :
  - Chaque zone doit avoir au moins un lieu et des PNJ associés

- Ajouter / enrichir le contenu :
  - Images / illustrations pour les zones et lieux clés
  - Fiches techniques PNJ (PV, CA, attaques, capacités spéciales, etc.)
  - Cartes simplifiées des lieux (même sous forme de schémas simples)
  - Suggestions de musiques d’ambiance par scène ou par lieu
  - Liens entre scénarios (branchements, conséquences des choix)

PHASE 5 – PERFORMANCE, PWA & EXPORT
================================================================================

Objectif : finition technique, utilisation hors-ligne et partage facilité.

5.1. Performance & tests
------------------------
- Lazy loading des vues :
  - Utiliser React.lazy et Suspense pour UniverseView, ScenariosView, etc.

- Optimisation des listes et calculs :
  - Utiliser useMemo et useCallback pour les grandes listes et la recherche

- Tests unitaires basiques :
  - Composants UI (Card, Tag, Button, etc.)
  - Hooks (useLocalStorage, useSearch)
  - Fonctions utilitaires (searchUtils, dataLinks, cn, etc.)

5.2. Hors-ligne, SEO & export
-----------------------------
- SEO (si déploiement public) :
  - Meta tags
  - Titres dynamiques par route (ex : “Univers – Ombre de la Spirale”)

- PWA :
  - Ajout d’un service worker
  - Manifest PWA
  - Mode hors-ligne pour les sessions sans internet (cache des données & assets)
